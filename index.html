Highly Available Web Application

Project entails the development of a cloud-based web application characterized by high availability and scalability. The architecture is designed to allow for independent scaling of the application server, database, and file server components. Amazon services such as VPC, EC2, RDS, and EFS are utilized in conjunction with load balancers and auto-scaling groups to efficiently handle traffic fluctuations and ensure scalability. Additionally, ElastiCache serves as a caching layer in front of RDS to optimize performance.


Overall, the project aims to deliver a robust and reliable web application infrastructure that can seamlessly adapt to varying workloads and demands. Furthermore, all resources have been provisioned using infrastructure as code methodologies.

  • VPC Configuration: Established a Virtual Private Cloud (VPC) with subnets distributed across two Availability Zones (AZs). This setup ensures fault tolerance and high availability for the hosted application.
  • Internet Connectivity: Implemented an Internet Gateway within the VPC to enable internet access for resources residing in the public subnets, facilitating communication with the outside world.
  • NAT Gateway: Deployed NAT Gateways in each public subnet to facilitate outbound internet connectivity for resources in the private application subnets, ensuring secure and controlled access to the internet.
  • RDS Aurora Multi-AZ Deployment: Configured a subnet group to support multi-AZ deployment of Amazon RDS Aurora. Created an Aurora DB cluster operating across the two data subnets, ensuring data redundancy and fault tolerance.
  • ElastiCache (Memcached): Set up an ElastiCache-Memcached server-side caching mechanism to alleviate database load by caching common queries, leading to improved performance and faster loading times for the application.
  • EFS Cluster: Established an Amazon Elastic File System (EFS) cluster to provide a shared filesystem for the web application servers, enabling seamless data sharing and access across multiple instances.
  • Application Load Balancer (ALB): Deployed an Application Load Balancer to distribute incoming traffic across the application servers hosted in the public subnets, ensuring optimal resource utilization and high availability.
  • Auto Scaling Group: Implemented a Highly Available (HA) application server setup using Auto Scaling Group with a launch template, allowing for dynamic scaling of application instances based on demand. This ensures reliability and scalability of the WordPress installation.
  • Terraform (IaC):Terraform was utilized to provision the infrastructure as code, ensuring consistency and repeatability in the deployment process. This allowed for the automated creation and management of the various resources, including the VPC, subnets, internet gateway, NAT gateways, RDS Aurora cluster, ElastiCache-Memcached, EFS cluster, Application Load Balancer, and Auto Scaling Group
