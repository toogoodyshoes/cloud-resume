Cloud Resume

The project is a sophisticated cloud-based web application built on AWS infrastructure, incorporating a range of services including Route 53, S3, CloudFront, Lambda, API Gateway, Amazon Certificate Manager, DynamoDB, and AWS Organizations. It serves as a showcase for my resume, featuring a visitor count functionality.


Employing HTML, CSS, and JavaScript for the frontend, and AWS Serverless Application Model (SAM) for the backend, the project adopts an Infrastructure as Code (IaC) approach for provisioning most of its infrastructure. This ensures efficient management and scalability of resources, making it a robust and professional solution for web application development on the AWS platform.

  • Domain Registration: Registered a domain name with a registrar and seamlessly integrated it with Route 53 for domain services.
  • Infrastructure Provisioning: Utilized AWS CDK (Infrastructure as Code) to provision resources on AWS, such as hosting the website on Amazon S3 and ensuring global delivery through CloudFront distribution.
  • Automation with CI/CD: Implemented a bash script along with Continuous Integration/Continuous Deployment (CI/CD) pipelines to automate the deployment process. This included deploying the website to Amazon S3 and invalidating the CloudFront cache.
  • Visitor Count Functionality: Integrated API Gateway as a proxy to trigger AWS Lambda functions, enabling real-time updates to the website's visitor count stored in DynamoDB.
  • SAM Infrastructure Deployment: Leveraged GitHub Actions CI/CD pipelines to seamlessly integrate and deploy the AWS Serverless Application Model (SAM) infrastructure.
  • GitHub Authentication: Established secure authentication between GitHub and AWS by registering GitHub OIDC as a federated identity in AWS Identity & Access Management (IAM), streamlining the GitHub Actions CI/CD process.
  • Additionally, AWS Cloud Development Kit (CDK) was employed to efficiently provision CloudFront, S3, and Route 53 resources, alongside the necessary IAM roles and policies. This streamlined the deployment process and ensured consistent and secure resource configuration across the AWS infrastructure.
